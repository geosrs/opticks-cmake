Raster Layer Properties

A raster layer is an image with each pixel representing one data value. The contrast and brightness of the image are based on the values selected in the Properties dialog. A raster layer is sometimes created as results from an algorithm plug-in.

The values can be displayed so the range of pixel values is stretched to fit the display range of the screen.

To open the Raster Layer Properties:

  1. Go to the Session Explorer window and select the image name.
  2. Right click. On the Image Menu, select Properties....The Properties window is displayed.

  1. When you're finished updating the properties:

To apply the new properties and close the dialog box, click the OK button.

To apply the new properties and leave the dialog box open, click the Apply button.

To cancel the changes and close the dialog box, click the Cancel button.

Display Configuration

  • Display Mode. Select grayscale or RGB.
  • Complex Component. This option is useful when loading complex data. It provides the means to select one of the four data display components of the data.
  • Opacity. Enter the opacity value.

Grayscale and RGB

The Grayscale and RGB sections enable you to select how the grayscale and RGB bands are displayed in the image.

  • Display Element. Enables you to select the raster data that is displayed. Different data can be displayed in each of the R, G, and B channels.
  • Display Bands. Enables you to select which bands are displayed. The data in the selected band depends on the parameters you select.

Opticks also provides three predefined color composites for image display: NIR, SWIR, and True Color. You can customize these composites. You can also create new composites.

To select one of the predefined composites, click the Presets button and select the name of the color composite.

For more information, see Color Composites.

  • Stretch Values. The stretch values are the range of values the stretch is mapped over. You can enter the Lower and Upper values for each band.
  • Stretch Units. The stretch units determines the meaning of the numbers displayed in the “stretch values” field. Select from the drop-down list.
  • Raw. The numbers reflect the same units as the spectral data. For example, if the data cube is in reflectance scaled from 0–10,000, the stretch values will correspond directly to these 0-10,000 values.
  • Percentage. The stretch values are represented in terms of their position relative to the maximum and minimum raw values in the data set.
  • Percentile. The stretch values are represented in terms of their percentile within the data set. For example, a stretch value that is higher than half the data values in the cube and lower than the other half is represented as “50.” A value that is higher than 75% of the data and lower than the other 25% is represented as “75.”
  • Standard Deviation. The stretch values are represented in terms of how many standard deviations they are from the average data value in the cube. A negative value indicates a stretch value that is less than the average and a positive value indicates a stretch value greater than the average.
  • Stretch Type. Use the drop-down list to select the stretch type.
  • Linear. The stretch is linear from the lower stretch value to the upper one. A value halfway from the lower to the upper (at the median of the two values) is shaded at 50% brightness.
  • Logarithmic. The contrast stretch is logarithmic in shape from the lower to the upper value. This has the effect of accentuating the weaker/darker features of the image.
  • Exponential. The contrast stretch is exponential in shape from the lower to upper value. This has the effect of suppressing weaker/darker features in the image and accentuating the brighter ones.
  • Histogram Equalization. Sets the raster layer to an equalized histogram contrast stretch. Every display value (0-255) has an equal number of pixels.
  • Favorites. Enables you to select, add, or remove favorites based on selected values.

Dynamic Texture Generation

  • Enable Dynamic Generation. This toggles the capability to dynamically generate textures for images. This is enabled only on systems with the hardware needed to support it.
  • Filter. Applies a filter to the data before it is displayed.
