Principal Component Analysis

Pixels often have similar data values in different spectral bands, especially for high spectral resolution data. Information extracted from the bands can be redundant or repetitive. To save time and memory space, it is useful to reduce or compress this repetitive data. Principal component analysis amplifies variations in the spectral data in order to reduce the amount of redundant data.

The Principal Component Analysis (PCA) is based on data statistics from the spectral bands. It can utilize the second moment, covariance, or correlation coefficient matrix. The principal components are basically coordinate transforms of the original data cube. Each component is a linear combination of the original band data. A new data cube is created containing the principal components from the original cube.

To perform a principal component analysis:

  1. On the General Algorithms Toolbar, select Principal Component Analysis. The Principal Component Analysis dialog box is displayed.

  1. Select the Transform method:
    • Calculate. When you select Calculate, you must also select the method used to calculate the transform coefficients.

The dialog defaults to the Second Moment matrix. Since Opticks provides a Second Moment Plug-In, its functionality is slightly different from the other two methods. If the Second Moment matrix is selected, the PCA Plug-In checks the active view for an existing second moment matrix. If one has already been generated, it is used to calculate the transform coefficients. If not, the PCA Plug-In launches the Second Moment Plug-In to generate the matrix. The covariance and correlation coefficient (the correlation coefficient matrix is a special case of the covariance matrix) matrices are generated by the PCA Plug-In and provide the same options as the Second Moment Plug-In.

    • Load from File. This allows you to load an existing set of transform coefficients from a file. The saved transform files have a file extension of “.pcaxxx” where xxx represents the statistical matrix used to calculate the transform coefficients: “.pcasmm” for second moment, “.pcacvm” for covariance, and “.pcaccm” for correlation coefficient.

Use the Browse button to browse to the file you want to use.

  1. Optional. Check the Region of Interest (ROI) checkbox and select the ROI from the drop-down list.

The ROI selection allows you to limit the area of the image over which the Principal Component Analysis will be performed. To select a region of interest, you must first create an “Area of Interest.” See AOI.

If ROI is not enabled, then the PCA transform will be applied to the entire loaded image.

  1. Opticks selects a default Number of Components. You can change this number. You can also select the number from a plot of the Eigen values. See Eigen Plot.

Most of the variation in the image will be contained in the lower numbered principal components. The higher numbered components will mainly contain noise. Thus there is little reason to generate the highest numbered components since they contain little useful information. The maximum number is limited to the number of loaded bands for calculating a new transform or the number of components saved if loading a transform from file.

  1. Select the Output Data Type from the drop-down list. Principal Component Analysis was originally developed for LANDSAT imagery, which contained unsigned one byte data. Many programs still only output PCA values as unsigned one-byte integers, regardless of the input data types. The drop-down list provides a full list of output data types.
  2. Opticks selects a default Scale Max Component Value. You can change this number.

The PCA component values will cover a wide range. To prevent loss of information due to floating point truncation during output, the PCA values are scaled so that the maximum PCA component value equals the maximum value for the output data type. The scale value defaults to the maximum value for the output data type, but you can change this value.

  1. Opticks selects a default Scale Min Component Value. You can change this number.

To prevent loss of information due to floating point truncation during output, the PCA values are scaled so that the minimum PCA component value equals the minimum value for the output data type. The scale value defaults to the minimum value for the output data type, but you can change this value.

  1. Click OK to begin the analysis.
  2. A dialog box may be displayed. This dialog box looks slightly different for each calculation type but it asks for the same information for each type. The example below is for Covariance.

Select either the Skip Factors or the AOI radio button.

If you select Skip Factors, you can enter the number of rows and columns that will be skipped when generating the image.

If you select AOI, the algorithm will generate components only from the AOI provided.

Click OK.

  1. In the Choose filename to save PCA Transform dialog box, Opticks automatically selects a file name and location for the saved file. You can change them.

Click the Save button to save the transform and continue, or click the Cancel button to bypass saving the transform and continue. Note that this Cancel button will not cancel the PCA; it will only prevent saving the transform.

If you have write permissions in the folder where the data resides, the transform is saved in the same directory as the original image with the file name of the original image plus “.pcaxxx” where xxx identifies the statistical matrix used in the analysis. If a file with that name already exists, you can overwrite the existing file, change the name, or leave the PCA transform in memory but not save the data.

  1. Click Save.

The new image is generated and displayed in the workspace area of the Opticks Main Application window.

Eigen Plot

If you selected to choose the number of components from the Eigen plot, the Eigen plot is displayed.

  1. Set the Number of Components to generate by editing the value or by clicking the plot near the highest desired component.
  2. To continuously update the number of components and the Eigen value for that component, hold down the left mouse while moving the mouse in the plot area.
  3. Click OK to set the number of components and continue.
