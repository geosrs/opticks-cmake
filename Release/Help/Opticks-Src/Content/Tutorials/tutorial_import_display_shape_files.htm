Tutorial: Import/Display Shape Files

Learning Objective: When finished with this tutorial, you will be familiar with importing and displaying shape files.

In this tutorial, you will import and display shape files and change the color of the displayed shape files in the following sequence:

  • Open a file
  • Import a shape file
  • Display a shape file
  • Change the color of the displayed shape file

Lesson: A shape file is a group of files that is used to exchange geographic information system (GIS) features between applications. GIS features are commonly used to represent roads, cities, rivers, lakes, and other geographic information. Overlaying GIS features in Opticks can help analysts get context of what is known to exist outside of their current data set. Shape files help exchange GIS information with other tools like ESRI’s ArcView product. ESRI is the developer of the shape file format, but many vendors support this widely accepted standard.

Shape files store GIS features in a vector-based format, this includes collections of points, multi-points, polylines, and polygons. Points can either be represented as a multi-point feature or a collection of single point features. Multi-point features are an unordered collection of points that represent a single entity, such as a group of wells that form a single unit. Polylines are a connect set of lines, which can be used to represent networks of roads, railroads, rivers, and other linear features. Polygons are used to represent an area, such as a lake or city boundary. A single shape file only supports GIS features of the same type. For example, a shape file that designates city roads would be made up of polylines, polygons, or points could not be part of the same shape file.

Opticks imports all shape files as annotations. Opticks currently has some limitations when loading shape files. Opticks does not support loading shape files in a coordinate system other than WGS84. Another limitation is that Opticks trims the shape file to fit boundary of the main data set. If the shape file isn’t within the extents of the current data set, no features will be loaded.

Note: Opticks only supports shape files that use the WGS84 coordinate system. Make sure that the shape files are created with this coordinate system if you plan to use them in Opticks.

Import File

  1. From the Opticks menu bar, select File, Import Data....
  2. Navigate to and select the file pyramids_psh_2feb2002_8bitgeotiff.tif. Click the Open button.

  1. The data set has georeferencing information that associates latitude/longitude values with each pixel. The image must be georeferenced. To activate the Geo Toolbar, right-click in the toolbar area and select Geo.
  1. On the Geo Toolbar, select Geo, Georeference.

  1. In the Georeference dialog box, accept the defaults and click OK.

In the Georeference dialog box, click the Close button.

A georeferencing layer is added to the data. Be sure your view is zoomed out far enough to see the markings.

Import a Shape File

  1. To activate the Geographic Features Toolbar, right-click in the toolbar area and select Geographic Features.
  2. On the Geographic Features Toolbar, go to the Geographic Features menu and select Import geodatabase....

  1. In the Opticks dialog box, select Shape file and click the Browse button.

  1. Navigate to and select the file Pyramids.shp. Click the Open button.

  1. Select the Display tab. Click the small yellow color box labeled Line Color. Choose custom. This will activate the Select Color window.

  1. In the Select color window, select the color red for the outline of the annotation features. Click OK.

  1. In the Opticks dialog window, click the OK button.

  1. In the Geodatabase import dialog box, click the Close button. The shape file is loaded, and the pyramids are now outlined in red.

  1. After the shape file is loaded, you can set up the display settings in more detail.

To activate the Annotation Toolbar, go to the Session Explorer. Right-click on the Pyramids.shp layer and select Activate.

  1. Select all the pyramids in the scene by clicking in the upper left corner of the scene and dragging a box around the entire image.

  1. On the Annotation Toolbar, click the Annotation Fill button and select blue.

This will fill the pyramids with the color blue. If desired, you can zoom in to see this more clearly.

You can also change line thickness and color. The text style and size do not affect the size of the feature’s label. The feature’s label color is the same as the line style.

In the example image, the GIS features overlay very well with the data set, although this is not always the case. How well shape files overlay with Opticks depends on the resolution of the source data and the shape file.

  1. Close all files.

Summary/Review: You should now be able to import and display a saved shape file and manipulate the colors of the displayed shape file.
