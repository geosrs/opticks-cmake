Tutorial: Export Shape Files

Learning Objective: When finished with this tutorial, you will be familiar with creating and exporting shape files.

In this tutorial, you will export shape files in the following sequence:

  • Open File
  • Georeference
  • Create complex polyline AOI
  • Export polyline shape file
  • Load shape file into ArcMap

Lesson: A shape file is a group of files that is used to exchange geographic information system (GIS) features between applications. GIS features are commonly used to represent roads, cities, rivers, lakes, and other geographic information. Overlaying GIS features in Opticks can help analysts get context of what is known to exist outside of their current data set. Shape files help exchange GIS information with other tools like ESRI’s ArcView product. ESRI is the developer of the shape file format, but many vendors support this widely accepted standard.

Shape files store GIS features in a vector-based format, this includes collections of points, multi-points, polylines, and polygons. Points can be represented as a multi-point feature or a collection of single point features. Multi-point features are an unordered collection of points that represent a single entity, such as a group of wells that form a single unit. Polylines are a connected set of lines, which can be used to represent networks of roads, railroads, rivers, and other linear features. Polygons are used to represent an area, such as a lake or city boundary. A single shape file only supports GIS features of the same type. For example, a shape file that designates city roads is composed of polylines. Polygons or points could not be part of the same shape file.

Opticks imports all shape files as annotations. Opticks currently has some limitations when loading shape files. Opticks does not support loading shape files in a coordinate system other than WGS84. Opticks may also trim or clip the shape file to fit boundary of the main data set. If the shape file isn’t within the extents of the current data set, no features are loaded. For information on setting how shape files are clipped upon import, see Import Shape File.

Note: Opticks only supports shape files that use the WGS84 coordinate system. Make sure that the shape files are created with this coordinate system if you plan to use them in Opticks.

Open File

  1. From the Opticks menu bar, select File, Import Data....
  2. In the Open dialog box, navigate to and select the file pyramids_psh_2feb2002_8bitgeotiff.tif. Click the Open button.

  1. After the file opens, you can view the data in color. Using shortcut keys, change the display to color as follows.

Click on the dataset and type 3r2g1bt.

“3r” sets band 3 as red, “2g” sets band 2 as green, “1b” band 1 as blue. The “t” shortcut key switches the display to color mode.

  1. This data set has georeferencing information.

From the Geo Toolbar, select Geo, Georeference.

  1. The Georeference dialog box is displayed. Accept the defaults and click the OK button to add a georeference layer to the data.

If necessary, adjust the Zoom Percentage so you can see the whole image. (Try adjusting it to 6%.)

Create a Polyline AOI

Only AOIs can be exported as shape files. AOIs can be lines, points, and polygons. However, all objects in the AOI are converted to points. To export AOI information as a polyline, do not add any points to the polyline while editing it.

  1. Go to the Session Explorer window. Select the pyramids image name. Right-click and select New Layer, AOI.

  1. From the AOI Pixel Selection Tool on the AOI Toolbar, select the polyline image.

  1. Using the Pan and/or Zoom In on Mouse functions, maneuver the image display to the GCP5 in the middle of the display window.

  1. In the Session Explorer, right-click on the AOI 1 layer and select Rename. Change the “AOI 1” layer name to “Roads.”

  1. Re-activate the AOI toolbar by selecting the Layer Edit Mode button on the Toolbox Toolbar.
  2. Using the left mouse button, select a point on the road. Move the mouse along the road and select another point using the left mouse button. Repeat these steps four times. To finish the polyline, double-click the last point.

Export Roads as Polyline Shape File

  1. In the Session Explorer, select the Windows tab.
  2. Navigate to and right-click on Roads. Select Export. In the Export dialog box, select go to the Exporter field and select Shape File Exporter. Click the Options button to open the Shape File Exporter dialog box.
  3. In the Shape File dialog box, go to the Base Name text box and change the name of the shape file from the default to pyramid_roads.shp.

In the Shape drop-down list, select Polyline.

Click the Add Feature button.

  1. In the Add Features dialog box, select AOI Element 1 and click the OK button to export the file.

  1. In the Shape File dialog box, click the OK button.

  1. In the Export Window, click Save.

Load shape file into ArcMap

  1. After saving the Roads shape file feature, load the file into ArcMap using the Add Data button or dragging the shape file from Windows Explorer into the main ArcMap view.

Point and Multipoint shape file features can be exported in the same way by selecting “Point” or “Multi-Point” from the “Shape:” drop-down menu on the export dialog. Add fields that describe each feature in more detail using the “Add field” button. For example, add a field called “Name” that contains the name of the road.

Although AOIs are the only layer type that can be exported as a shape file, some layer types can be converted to AOIs. For example, a result matrix can be converted to a set of AOI pixels and exported as a group of point or multi-point shape file features.

  1. Close all files.

Summary/Review: You can now create, save, and export a shape file ready to be imported into ArcMap using many of the common core tools in Opticks.
