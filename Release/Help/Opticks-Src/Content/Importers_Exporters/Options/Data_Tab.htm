Data Tab

The Import Options Data tab provides basic import data information such as file name and type. Depending on the type of data file, it also provides more detailed information such as the number of rows and columns, pixel size, and display mode. The example below provides detailed information. Note that you can expand the Units field to display more units information.

Some fields on this tab are display only. For example, the "Name" field is display only. Other fields such as the "Bad Values" field, the pixel size fields, the units field, and the bands fields can be edited. To edit a field, double-click in the Value cell next to the item you want to edit. If the value can be edited, the cell is activated or a drop-down is displayed.

When you click the Set Display Bands button, a menu appears that enables you to select various combinations of displayed bands based on wavelengths. Opticks provides three predefined color composites for image display: NIR, SWIR, and True Color. You can customize these composites. You can also create new composites. For more information, see Color Composites.

Performance implications of Interleave Format and Processing Location

Interleave Format has three values you can select: BIL, BIP, and BSQ. This specifies the interleave of the data when loaded. Each value may have performance implications depending on what processing you will do with the data and what Processing Location you select. Most algorithms are tuned for either BIP or BSQ access and will perform best on data loaded with the same interleave. The Opticks Band Math and Principal Component plug-ins are tuned for BIP access. The Opticks band-statistics and image rendering are mildly tuned for BSQ and BIL access but this tuning is only pronounced if the Processing Location is On Disk or On Disk (Read Only).

Processing Location has three values you can select: In Memory, On Disk (Read Only), and On Disk. Each of these has performance implications.

  • In Memory: This forces the application to allocate a single block of memory large enough to hold the entire data cube and to load the data cube into it using the interleave specified by the user. If the computer has sufficient memory to hold the entire data cube, this is typically the best option. The cube will take a bit longer to appear initially, but will perform better during all subsequent manipulation and processing.
  • On Disk (Read Only): This pages the data in from the data file as needed. It uses less memory than the In Memory option, but forces the data to be paged in each time it is needed. Not all importers support this option. In addition, some algorithms want to modify the data cube in place and cannot do so on data loaded using this option. This is typically the fastest option for the initial display of the image, but will be the same as On Disk and slower than In Memory for subsequent access to the data. This option also forces you to select the same Interleave Format on the “Data” tab as on the “File” tab (no re-interleaving on load).
  • On Disk: This makes a copy of the data file on disk and then pages the data in from the copy as needed. Its memory usage is the same as the On Disk (Read Only) case, but without the importer, algorithm, and interleave restrictions. It will take longer for the image to come up initially than the On Disk (Read Only) option, since a copy of the data file will be made during load. The copy will also take up space on the your disk drive while the data cube is loaded.

Note that re-interleaving on import (using a Interleave Format on the “Data” tab that is different from the value on the “File” tab) with In Memory or On Disk will result in a small delay in the initial display of the image because the data needs to be re-organized during load or during the creation of the copy of the data file. Re-interleaving on import is not possible with On Disk (Read Only).
